# TTS 听书功能重构设计

**日期：** 2026-06-20
**状态：** 待审批

## 问题概述

1. **关闭弹窗停止语音** — 当前 `showTTS` 同时控制弹窗可见性和播放状态，关闭弹窗时直接调用 `stopTTS()` 清空音频和队列
2. **朗读位置错误** — EPUB 文本提取 API 无效（已修复 `epubCurrentContents`），且 TTS 只读当前页，不会自动翻页继续

## 参考产品

微信读书、喜马拉雅、Apple Books — 均采用"完整播放器面板 + 悬浮迷你播放条"模式

## 设计方案：悬浮迷你播放条

### 1. 状态模型重构

将弹窗 UI 和播放状态解耦：

| 变量 | 类型 | 用途 |
|------|------|------|
| `showTTSPanel` | `boolean` | 控制完整设置弹窗的显示/隐藏 |
| `ttsPlaying` | `boolean` | 音频是否在播放（独立于弹窗） |
| `ttsPaused` | `boolean` | 是否暂停 |
| `ttsQueue` | `string[]` | 待播放句子队列 |
| `ttsIndex` | `number` | 当前播放到第几句 |
| `currentAudio` | `Audio` | 当前音频实例 |

**关键行为变更：**
- 点击"听书"按钮 → 打开弹窗 + 开始播放
- 关闭弹窗（X 或遮罩）→ **只隐藏弹窗，不停止播放** + 底部出现迷你播放条
- 点击迷你播放条 → 重新打开完整弹窗
- 点"停止"按钮 → 停止播放 + 隐藏迷你条
- 组件卸载（`onBeforeUnmount`）→ 停止播放（已有逻辑，保持不变）

### 2. 迷你播放条 UI

**位置：** 阅读器底部，`position: fixed; bottom: 0`，悬浮在阅读内容上方

**布局：**
```
┌─────────────────────────────────────────────┐
│ 🔊 AI 听书中 · 第 3/12 句    ▶暂停  ■停止 │
└─────────────────────────────────────────────┘
```

- 左侧：`Volume2` 图标 + "AI 听书中" + 当前句/总句数
- 右侧：暂停/继续按钮 + 停止按钮
- 高度：48px
- 背景：白色，顶部 1px 阴影（`shadow-sm`）
- 点击整条（不含按钮区域）→ 打开完整弹窗
- 过渡动画：底部滑入/滑出（`translate-y` + `transition`）

**阅读区域适配：**
- 当迷你条显示时，阅读器 footer 和内容区加 `padding-bottom: 60px` 避免遮挡

### 3. 完整弹窗调整

- 弹窗不再有"停止"按钮的功能绑定（停止统一在迷你条和弹窗内都有）
- 关闭弹窗（X / 遮罩点击）→ `showTTSPanel = false`，**不调用 `stopTTS()`**
- 弹窗内保留：音色选择、语速调节、进度条、暂停/继续/停止/重新播放按钮
- 播放状态显示保持不变（"正在播放 X / Y 句" + 进度条）

### 4. 自动翻页连续朗读

**触发时机：** `playNextTTS()` 检测到 `ttsIndex >= ttsQueue.length` 时

**逻辑：**
1. 判断当前格式（epub / txt / pdf）
2. 如果不是最后一页/章 → 调用对应的翻页函数（`epubNext()` / `txtNext()` / `pdfNext()`）
3. 等待页面渲染完成（`nextTick` + 短延时）
4. 提取新页面文本 → 分句 → 追加到 `ttsQueue`
5. 继续播放
6. 如果是最后一页 → 调用 `stopTTS()`，toast "朗读完毕"

**队列管理：** 滚动窗口 — 当队列超过 100 句时，丢弃前 50 句（已播放过的），调整 `ttsIndex`。避免长书内存膨胀。

### 5. 文本提取（已有修复 + 增强）

- **EPUB：** 通过 `epubCurrentContents` 获取当前页 DOM 文本（已修复）
- **TXT：** 通过 `calcTxtPageSize()` + `txtPage` 计算当前页文本（已有）
- **PDF：** 通过 `extractPdfText()` 提取当前页文本（已有）

自动翻页时，翻页后重新提取新页面文本并追加到队列。

### 6. 文件变更清单

| 文件 | 变更 |
|------|------|
| `frontend/src/views/Reader.vue` | 状态模型重构、迷你播放条组件、关闭逻辑修改、自动翻页逻辑、padding 适配 |

> 本次重构集中在 `Reader.vue` 一个文件内，不新增组件文件。迷你播放条作为 Reader 内的局部模板，保持简单。如后续 TTS 功能扩展（如全局播放、后台播放），再抽取为独立组件。

### 7. 边界情况

| 场景 | 处理 |
|------|------|
| 最后一页读完 | 停止播放，toast "朗读完毕" |
| 翻页失败 | toast 错误提示，停止播放 |
| 页面切换时正在播放 | 音频继续，下一句用新页面文本 |
| 组件卸载 | 停止播放，清理资源（已有逻辑） |
| 未登录 | toast "请先登录"（已有逻辑） |
| 文本为空 | toast "没有可朗读的文本"（已有逻辑） |
